# Prompt: Transformar Pull Request en Comentario de Jira

En base a un Pull Request de GitHub, genera un comentario estructurado para Jira siguiendo el formato DoD Summary. Solo debes poner los puntos clave, de forma descriptiva, no tan técnica como en la PR.

## Formato de Salida Esperado

```markdown
## DoD Summary

- **Summary of what was done:**
    - [Descripción principal del cambio y su propósito]
        - **Database changes:** [si aplica]
            - [Lista de cambios en base de datos]
        - **API Changes:** [si aplica]
            - [Lista de cambios en API]
        - **Frontend/UI changes:** [si aplica]
            - [Lista de cambios visuales o de interfaz]
        - **Other changes:** [si aplica]
            - [Otros cambios relevantes]
- **Code review:** PR #[número] - [título]  
- **Validation:**
    - For manual testing:
        ```[lenguaje]
        [Ejemplos de uso o comandos de prueba]
        ```
    - For automated testing:
        - [Instrucciones para pruebas automatizadas]
        - [Ubicación de tests o Bruno collections]

## Rollout

- **PR for [ENVIRONMENT]:**
    - [Link al PR #número]
```

## Ejemplo de Referencia

```markdown
## DoD Summary

- **Summary of what was done:**
    - New `sending_workcenter` field for category management to specify a single origin/source workcenter, separate from destination workcenters. The field is optional and fully backward compatible.
        - **Database changes:**
            - New table `ShiftBookCategorySendingWC`
            - Association `sendingWorkcenter` added to `ShiftBookCategory`
        - **API Changes:**
            - New parameter `sending_workcenter: CategoryWCInput` in create/update actions.
            - OData support via `$expand`
            - Returns HTTP 409 Conflict for duplicate workcenters
            - Bruno Collection updated with new changes
- **Code review:** PR Shared  
- **Validation:**
    - For manual testing:
        ```http
        POST /ShiftBookService/createCategoryWithDetails
        {
          "werks": "2000",
          "sending_workcenter": {"workcenter": "WC_ORIGIN"}
        }

        POST /ShiftBookService/updateCategoryWithDetails
        {
          "category": "uuid",
          "werks": "2000",
          "sending_workcenter": {"workcenter": "WC_NEW"}
        }

        GET /ShiftBookService/ShiftBookCategory?$expand=sendingWorkcenter
        ```
    - For automated testing, use Bruno collection:
        - From path `__apiCollection/ShiftBookCategory Management Actions/`
            - Create Category with Sending Workcenter.bru
            - Get Category with Sending Workcenter.bru
            - Update - Remove Sending Workcenter.bru
            - Test Duplicate Sending Workcenter (409).bru

## Rollout

- **PR for MANU-DEV:**
    - [MR #150](https://gitlab.com/syntax-cloud/sap-products/manu/dm/shift-book/-/merge_requests/150)
    - [MR #151](https://gitlab.com/syntax-cloud/sap-products/manu/dm/shift-book/-/merge_requests/151)
```

## Instrucciones Clave

1. **Enfoque descriptivo, no técnico**: Explica QUÉ se hizo y PARA QUÉ, no tanto el CÓMO técnico
2. **Organiza por tipo de cambio**: Database, API, Frontend, etc.
3. **Incluye validación práctica**: Ejemplos de uso real con código/comandos
4. **Backward compatibility**: Menciona si los cambios son retrocompatibles
5. **Links relevantes**: Incluye enlaces al PR y recursos relacionados
