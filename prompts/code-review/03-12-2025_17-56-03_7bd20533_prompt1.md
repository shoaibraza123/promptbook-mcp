## Content

Genera un prompt para transformar un Pull Request de GitHub en un comentario estructurado de Jira siguiendo el formato DoD (Definition of Done) Summary.

El prompt debe:
- Extraer información clave del PR (título, cambios, archivos modificados)
- Crear secciones estructuradas para Jira
- Incluir contexto técnico relevante
- Mantener formato profesional
- Ser conciso pero completo

Ejemplo de estructura:
## DoD Summary
- **Summary of what was done**: [descripción breve]
- **Technical details**: [cambios técnicos]
- **Impact**: [qué afecta]
- **Testing**: [cómo se verificó]
